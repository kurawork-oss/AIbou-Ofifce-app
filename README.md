# AIbou Office — AIだけで回る会社

AI社員が営業・事務・マーケティングを自律的に回すバーチャルカンパニー管理アプリ。
あなた(代表)の仕事は、AI社員から上がってくる申請を**許可 / 未許可**するだけ。

## 3つのモード

| モード | 内容 |
|---|---|
| 🏢 **オフィス** | **3Dオフィス**(銀河背景・トイ調・高解像度):**ロボット社員とトモコレ風の人間社員**が混在(それぞれ個性つき)。エントランスから出社し、通路や会議室のドアを通って歩いて移動する。ロボット/デスク/部署ホワイトボード/共有キャビネット/会議室モニターを**クリックすると詳細パネル**(成果物・目標進捗・取扱商材・議事録)が開く |
| 📥 **確認事項** | AI社員からの申請ボックス。社員追加提案・ツール利用申請・経費申請を許可/未許可で処理 |
| 👥 **社員管理** | 名前・役職・部署・現在のステータス・Googleメールアドレス・パスワードの一覧管理(インライン編集可) |

## 会社の仕組み(シミュレーションエンジン)

3秒ごとのtickで会社が自律的に回ります。

```
マーケティング部        事務部                営業部
業界リサーチ ──知見──▶ 営業リスト作成 ──リスト──▶ 架電・メール営業 ──▶ アポ獲得 🎯
無料発信(X/note/YouTube)  問い合わせ対応         提案ブラッシュアップ
        └──────────── 定例MTG(自動開催・議事録生成)────────────┘
```

- **リサーチ知見** … マーケが生産。事務のリスト精度と営業の提案の質を上げる
- **営業リスト** … 事務が生産。営業が消費してアポどり(戦略レベルが高いほどアポ率UP)
- **定例MTG** … 各部署から1名参加で自動開催。決定事項が戦略レベルに反映
- **休憩** … 仕事の在庫がない社員は休憩スペースでゲーム。仕事が入ると即復帰
- **申請** … 社員は3つの注意事項(①生産性最優先 ②新ツールは申請 ③経費は無料代替を検討のうえ申請)を守って代表に申請を上げる
- **社員追加提案** … 業務が逼迫するとAIが増員を提案。許可時にその社員用のGoogleアカウント(メアド・パスワード)を登録して入社

## セットアップ

```bash
npm install
npm run dev   # http://localhost:3000
```

会社の状態はブラウザのlocalStorageに保存されます(「会社をリセット」で初期化)。

### LLMによる議事録生成(任意)

定例MTGの議事録を現在のKPIに基づいてLLMが生成します。優先順位は **Gemini(無料枠あり)→ Claude → テンプレート** で、キー未設定でもテンプレートで完全動作します。

```bash
# .env.local(.env.example 参照)
GEMINI_API_KEY=AIza...        # Google AI Studio・無料枠あり
ANTHROPIC_API_KEY=sk-ant-...  # フォールバック(従量課金)
```

### Supabase 記憶保管庫(任意)

1. Supabaseで無料プロジェクト作成 → SQL Editorで `supabase/schema.sql` を実行
2. `.env.local` に `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY` を設定

有効になると:
- 各AI社員の活動が**Googleメールアドレスに紐づく記憶保管庫**(`employee_memories`)へ蓄積
- 会社スナップショット(社員・KPI・議事録)が `company_state` へ定期保存
- ヘッダーに「☁️ Supabase同期中」バッジが表示

## オフィス内で見れるデータ(クリック操作)

- 🗄️ **共有キャビネット** … 全社員の成果物(営業リスト・リサーチメモ・提案資料・議事録・アポ獲得報告)を部署フィルタ付きで閲覧。スプレッドシートは中身の表まで展開
- 🤖 **ロボット社員 / デスク** … 本人のプロフィール・現在の業務と進捗・Googleアカウント・本人が作ったデータ一覧
- 📋 **部署ホワイトボード** … 部署のミッション・目標と進捗バー・直近MTGの決定事項・メンバーの稼働状況
- 🤝 **会議室モニター** … 進行中/直近MTGの発言ログと決定事項

## 🏢 事業部(商材ごとの独立チーム)

商材=事業部で、**各事業部が独立した営業・事務・マーケのチーム**を持ちます。

- ヘッダー「⚙️ 会社設定」で会社名と商材(事業部)を設定。商材ごとに**営業対象(法人B2B / 個人B2C / 両方)** を選択
- 「+ 事業部を追加」で商材を増やすと、その事業部に**スターターチーム(営業・事務・マーケ各1名)が自動配属**され、独立して稼働を開始
- KPI(アポ・リスト・知見など)も**事業部ごとに独立**して集計。ヘッダーのKPIは全社合算
- オフィス上部の**事業部スイッチャー**で切り替えると、その事業部だけのオフィス(社員・成果物・議事録・目標)が表示され、見やすく操作しやすい
- 成果物やホワイトボードもその事業部の商材・対象(法人=社名/個人=氏名)を反映

**オフィス操作**: ドラッグで回転、ホイールで拡大、右上の ＋/－/🎯(視点リセット)ボタン。

## 技術構成

- Next.js (App Router) + TypeScript + Tailwind CSS
- three.js + react-three-fiber + drei(3Dオフィス)
- zustand(状態管理・localStorage永続化)
- Gemini API(REST)/ Anthropic TypeScript SDK(議事録生成・任意)
- Supabase(記憶保管庫・任意)

## ⚠️ パスワードの取り扱い

社員管理のGoogleアカウント情報はブラウザ内(localStorage)にのみ保存されますが、平文です。実運用ではパスワード管理ツールの利用と、アプリへの実パスワード保存の回避を推奨します。
